## 背景介绍

有一个C++项目，从web前端接收操作命令，根据命令执行不同的业务操作
以下是C++程序示例：

```C++
sw::redis::ConnectionOptions conn_opts;
conn_opts.host = redis_host;         // 主机地址
conn_opts.port = redis_port;         // 端口号
conn_opts.password = redis_password; // 密码（包含特殊字符也没问题）
conn_opts.socket_timeout = std::chrono::milliseconds(1000); // 设置 socket 超时，确保 consume() 有超时

auto redis = sw::redis::Redis(conn_opts);

// 创建订阅者
auto sub = redis.subscriber();

// 设置消息回调
sub.on_message([](std::string channel, std::string cmd)
{
    json j = json::parse(cmd);
    string cmdName = j["cmd_name"];
    string cmdPara = j["cmd_para"];

    //根据cmdName执行不同的操作
    if (cmdName=="command1")
    {
        // do something
        // 进一步解析cmdPara
    }
    else if  (cmdName=="command2")
    {
        // do something
        // 进一步解析cmdPara
    }
    ...
});

// 订阅操作命令消息频道
sub.subscribe("operation_cmd");

// 接收前端操作命令消息
while (running)
{
    sub.consume();
}
```

现在要求本项目能够将前端画面的用户操作命令通过websocket传递到后端，然后通过redis主题"operation_cmd"发布出去

## 新增功能需求

- 检查packages\shared\src\redis_types.ts里命令类型CmdPushMessage的定义是否合理
- 在前端项目创建命令发送函数
- 在后端项目创建命令接收及发布到redis主题"operation_cmd"的功能

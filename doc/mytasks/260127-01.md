# 为项目的前后端增加WebSocket通讯功能，要求：
- 前端具备连接断开后能够自动重连的功能
- 前端所有页面共享一个WebSocket连接
- 前端订阅数据，后端推送数据
- 在views/HealthCheckView.vue页面增加功能，不要删除现有功能，增加的功能是：在页面加载的时候通过WebSocket连接发送一个订阅请求，内容是当前页面请求后端向前端推送的数据项名称(tag)的集合，数据格式是一个字符串数组，每个字符串代表要订阅的数据项名称(tag name)，此页面订阅的数据项是tag1、tag2和tag3
- 后端向前端推送的数据是JSON字符串，parse后的格式为：{ tag: string; value: string }，其中tag是数据项名称，value根据不同的tag解析为不同的JSON对象，不同的JSON对象用typescript类型分别定义，tag1的数据类型是{ ph: string; lh: string; czh: string; tlxh: number; zzwj: number; zzbh: number }, tag2的数据类型是number, tag3的数据类型是长度为6的number数组，请用typescript定义上述类型
- 前端用Pinia store管理后端推送的数据，可以根据数据项名称从Pinia store获取对应的数据
- 在views/HealthCheckView.vue页面增加一个table组件，只有一行，显示tag1的6个字段；增加一个input组件显示tag2,增加一个table组件，6行1列，显示tag3的number数组
- 后端需保存每一个WebSocket连接所订阅的数据项集合，当收到订阅请求后，需用此请求包含的数据项集合替换原来订阅的数据项集合，强调一下，是整体替换而不是追加数据项。另外，当连接断开后，此连接被移除，其订阅的数据项集合也要清空
- 后端开发一个数据模拟测试功能，将tag1初始化为{ ph: '123456'; lh: '12345678'; czh: '3456'; tlxh: 1; zzwj: 139.7; zzbh: 5 }，tag2初始化为10，tag3初始化为[1 2 3 4 5 6]，然后每秒钟将tag1的tlxh字段值加1，tag2的值加1，tag3数组每一项的值加1，并查询每个WebSocket连接订阅的数据项集合，如果订阅了tag1、tag2和tag3则向前端推送数据。
